# Sources

Source records should make data provenance visible.

## Source Types

- `vendor`: official vendor/manufacturer page.
- `docs`: official technical documentation.
- `benchmark`: reproducible benchmark result.
- `community`: community submission.
- `app_seed`: value migrated from the current application.
- `estimate`: calculated or inferred value.

## Promotion Rules

- `app_seed` data starts as `estimated` or `community`.
- `vendor` data can become `official` after URL and units are reviewed.
- `benchmark` data can become `verified` only when methodology is clear.
- Conflicting data should remain visible until maintainers resolve it.
